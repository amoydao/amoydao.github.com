用hexo+GitHub快速搭建个人博客

发表于 2015-10-01   |   分类于 器   |  

开始之前：在GitHub上建立博客空间

  • 注册GitHub账号：访问github.com，注册登录
  • 创建博客空间：
    GitHub Pages分两种，一种是你的GitHub用户名建立的username.github.io这样的用户&组织页（站），另一种是依附项目的pages。想建立个人博客是用的第一种，每个用户名下面只能建立一个。
    1.登录到GitHub的个人主页，右上角处点击「+」—>「New repository」。

    2.填写你的项目名，如cnyod.github.io，和描述信息（可选），点击「Create Repository」 完成创建。

一、环境准备

1.程序安装

  • 安装 git
  • 安装 node.js
  • 安装 sublime text
  • 安装 hexo（运行Git Bash） ：
    $ npm install -g hexo-cli

2.配置SSH keys公钥

运行Git Bash，建立本地与远程的github的联系

  • 设置用户名密码
    $ git config --global user.email  "amoydao@qq.com"    #你的邮箱
$ git config --global user.name  "amoydao"   #你的用户名
  • 生成密钥
    $ ssh-keygen -t rsa -C "bu.ru@qq.com"   #你的邮箱
    然后系统会出现一些代码，按回车键，使用默认配置就好。
    Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):  #回车就好
    然后系统会要你输入你的GitHub密码：
    Enter passphrase (empty for no passphrase):  #输入密码
Enter same passphrase again:  #再次输入密码
    注意：输入密码的时候没有*字样的，你直接输入就可以了。
    最后看到这样的界面，就成功设置ssh key了：
  • 添加SSH Key到GitHub
    在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。
    1）到C:/Users/your_user_directory/.ssh（比如C:\Users\amoydao.ssh）中，用文本编辑工具打开id_rsa.pub文件，全选复制里面的代码。如果看不到这个文件，你需要设置显示隐藏文件。
    2）登录到GitHub的个人主页，右上角处点击「设置按钮/Settings」—>选择「SSH keys」—>点击「Add SSH Key」，把复制的内容粘贴进去，然后点击Add Key按钮即可。

  • 测试一下
    在Git上输入下面的命令，看看设置是否成功，git@github.com的部分不要修改：
    $ ssh -T git@github.com
    系统会反馈：
    The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?
    输入yes就好，然后会看到：
    Hi <em>username</em>! You've successfully authenticated, but GitHub does not provide shell access.
    配置成功，你已经可以通过SSH链接到GitHub了。

二、安装HEXO

  • 新建博客文件夹：英文路径，可建在U盘里。
  • 安装hexo（在该文件夹上右键运行Git Bash）
    $ npm install hexo --save
$ hexo init 
$ npm install
  • 生成预览
    $ hexo s
  • 访问 http://localhost:4000

三、安装主题

  • 挑选模板：有那些好看的hexo主题?
  • 使用主题：iissnan/hexo-theme-next
  • 克隆安装：
    $ git clone https://github.com/iissnan/hexo-theme-next themes/next
  • 启用主题：打开 站点配置文件_config.yml，找到 theme 字段，并将其值更改为 next。
  • 生成预览：
    $ hexo s
  • 访问 http://localhost:4000 ，确保站点正确运行

四、设置主题/站点/插件

参考：5分钟快速安装

五、发布到Github

  • 部署前需要配置_config.yml文件。
    deploy:
type: github
repository: https://github.com/amoydao/amoydao.github.io.git  #在第二步中你创建的GitHub仓库(repository)路径
brach: master
  • 执行指令完成部署，注意部署会覆盖掉你之前在版本库中存放的文件。
    $ hexo clean  # 清空
$ hexo g  # 本地部署，生成静态页面至hexo\public\目录
$ hexo s # 启动本地服务，进行文章预览调试，http://localhost:4000
$ hexo d # 发布到GitHub上，需要根据提示输入账号和密码
    注意：命令必须在init目录下执行，否则不成功，但是也不报错。
  • 可能会出现该问题
    **hexo ERROR**: Deployer not found: github
    解决方法：
    1.安装 npm install hexo-deployer-git –save
    2.将deploy 的 type由github改为git

六、个人域名

1. 购买域名

  • 购买国内域名商的域名需要备案，据说还各种限制，就根据推荐上国外的Godaddy购买，可以使用支付宝。
    搜你想要的域名—>根据引导完成支付，详细操作就不一一述说。附加收费服务不要管。在支付时可以上网搜godaddy的优惠码，省点钱。

2. 绑定域名

  • 设置Dnspod
    流传Godaddy的域名解析服务器被墙掉，导致域名无法访问，因此需要把域名解析服务迁移到国内比较稳定的服务商处，这个迁移对于域名来说没有什么风险，最终的控制权还是在Godaddy那里，你随时都可以改回去。
    这里选择DNSPod的服务。
    1）登录注册并添加域名记录，可参考DNSPod的帮助文档：https://www.dnspod.cn/Support
    2）在DNSPod自己的域名下进行设置，如下图：

    两条A记录：192.30.252.153和192.30.252.154
    一条cname记录：你在github注册的仓库
    注意：如博客不能登录，有可能是github更改了空间服务的ip地址，记得及时到在GitHub Pages查看最新的ip即可。
  • 设置 GitHub
    需要在博客文件根目录\source下，新建一个命名为CNAME的文件（不带后缀），在里面写你要绑定的独立域名，如cnyod.com，并用Git部署到GitHub上。
    建议参考github pages关于域名绑定的帮助文档，先在GitHub上直接创建CNAME文件，再下载放置在source文件夹里。
  • 设置Godaddy
    登录Godaddy—>点击管理我的域名

    修改Nameservers为这两个地址：f1g1ns1.dnspod.net、f1g1ns2.dnspod.net。

    详细可以参考Dnspod的帮助文档：
    Godaddy注册商域名修改DNS地址

七、写文章

  • 新建文章
    在hexo文件夹上打开Git，执行如下命令，发表新文章。

    $ hexo n "postName"   #新建文章。postname文章文件的命名，不是文章标题。

    执行命令后，会在项目\source_posts中生成my new post.md文件，用编辑器打开编写即可。
    当然，也可以直接在\source_posts中新建一个md文件。
    注意：建议用Sublime Text编写md文件，否者默认的文本编辑器可能会导致中文乱码现象。

  • 用Markdown语法写文章
    建议按此标准语法写：

    title: postName #文章页面上的显示名称，可以任意修改，不会出现在URL中
date: 2013-12-02 15:30:16 #文章生成时间，一般不改，当然也可以任意修改
categories: example #分类
tags: [tag1,tag2,tag3] #文章标签，可空，多标签请用格式，注意:后面有个空格
description: #附加一段文章摘要，字数最好在140字以内。
--- 
#以下正文

关于MD语法，请看这里：
[1] 献给写作者的 Markdown 新手指南
[2] Markdown语法说明
推荐使用简书写文章，支持Markdown语法，并可实时在线预览，非常方便。
我是先在简书上写完后，再把内容直接复制到Hexo的MD文件里发布。

  • 上传文章
    写完后，在Hexo文件夹右键打开Git，执行如下命令，推送到服务器上：
    hexo clean  #清理一下本地文件，以免出现一些诡异的情况
hexo g  #生成在本地
hexo d  #部署到GitHub
